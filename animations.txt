How to map these to simple CSS-only implementations (short recipe)

Simple fades: transition: opacity .3s ease; initial .hidden {opacity:0;pointer-events:none} .shown {opacity:1}
Slide-in: transform: translateX(-20px); opacity:0 → transform:none; opacity:1; transition: transform .5s ease, opacity .5s ease
Reveal-on-scroll: IntersectionObserver toggles a class that triggers transitions (pure CSS cannot detect scroll-in-viewport reliably)
Staggered reveals: use CSS custom property for --delay and inline style="--delay:.12s" with transition-delay: var(--delay)
Shake/bounce: keyframes targeting transform: translateX/Y with easing steps
Morphs (bubble/polyfold): use SVG path morphing (SMIL or JS) or CSS border-radius + scale; realistic morphs usually need JS or Lottie
Particles: use canvas or many absolutely-positioned .particle elements animated with keyframes for speed/opacity
Filmstrip: animate background-position or use CSS sprite/frame-index steps with steps() timing function
